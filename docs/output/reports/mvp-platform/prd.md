# PRD — 乡旅e模式 Wave 1 MVP（功能页演示）

## 1. 问题与目标

**问题**：旅游信息分散、路线安排不清晰、每日行程难快速规划。

**目标**：交付可演示的轻量旅游路线规划 Web 系统，形成完整闭环：

注册登录 → 按天数/主题/风格筛选 → 结果列表 → 路线详情 → 关键词搜索 → 用户反馈。

## 2. 范围（In）

| 功能 | 验收 |
|------|------|
| 注册 / 登录 / 登出 | 用户名唯一；密码不明文；顶栏显示用户态 |
| 首页智能推荐表单 | 计划天数、兴趣主题、规划风格三下拉 + 提交 |
| 推荐结果 | 路线卡片；无结果态；相关推荐 ≤3 |
| 路线详情 | 概述、每日安排、餐饮/住宿/参考成本 |
| 关键词搜索 | 仅路线；顶栏有入口 |
| 反馈 | 文本提交 + 成功提示 |

## 3. 范围（Out）

AI 对话、电商预订、支付、管理后台、地图全屏、APP。

## 4. 地域与数据

- **全国范围**乡村旅游 / 自然观光 / 亲子休闲 / 文化体验场景。
- 种子路线按**区域类型模板**组织（江南水乡、西南梯田、华北河谷、徽州山地等），**不**绑定单一城市景点库。
- 兴趣主题枚举：文化体验 · 自然观光 · 亲子休闲 · 乡村旅游（及扩展）。
- 规划风格：经济型 · 舒适型 · 豪华型。
- 计划天数：1-2 · 3-5 · 6-7。

## 5. 技术

- 前端：`frontend/app/` HTML/CSS/JS 功能页演示（可先本地种子数据）。
- 后端：`backend/` Flask + SQLAlchemy + Bcrypt + SQLite REST（与 ADR-0002 一致；可与前端演示并行）。
- 品牌文案：任务书 **乡旅e模式**；历史文档「乡海云途」作别名保留时可互指。

## 6. 页面

P1 首页 `/` · P2 登录 · P3 注册 · P4 智能推荐结果 · P5 搜索 · P6 详情 · P7 反馈 + 全局壳。

## 7. 成功标准

现场可演示主流程；文档不写未实现功能；数据支持多主题多天数筛选。
