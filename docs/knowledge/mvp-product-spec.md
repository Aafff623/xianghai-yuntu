# MVP 与产品规划 — 核对清单

> 从 `docs/source/`、`backup/`、`coding-legacy/` 提炼的**可开发事实**。  
> 竞赛文档中的商业/财务/远景架构单独标注，**不进入 Wave 1 MVP**。

最后核对：2026-07-10

---

## 1. 产品定位（项目书）

| 项 | 内容 |
|----|------|
| 品牌 | **乡海云途**（对外统一；文档亦见「乡旅与海 e 模式」） |
| 形态 | B/S **Web 平台 1.0**（App 为远期，APP 组企划仅「搭建优化」） |
| 地域 | 山东沿海 **7 市**：青岛、烟台、威海、日照、东营、潍坊、滨州 |
| 场景 | 海洋 + 乡村旅游；另有一条 **海大校园版** 支线（四校区路线） |
| 痛点 | 信息分散、服务同质化、产销断层、信息不对称 |
| 目标用户 | 城市中高收入家庭、亲子研学、年轻背包客（见 §2） |

## 3. 功能分层（开发边界）

### Wave 1 — MVP（必须做，对齐 legacy + 项目书 §3.3.2）

| # | 功能 | 说明 | legacy 对照 | 状态 |
|---|------|------|-------------|------|
| 1 | 注册 | 用户名唯一、密码确认、Bcrypt | ✅ | 待开发 |
| 2 | 登录 / 登出 | Session；顶部显示用户名 | ✅ | 待开发 |
| 3 | 首页 + 智能推荐表单 | 天数 / 类型 / 预算三选一提交 | ✅ 首页含表单 | 待开发 |
| 4 | 智能推荐结果 | 按 `day_range` + `route_type` + `budget_level` 过滤 | ✅ | 待开发 |
| 5 | 相关推荐 | 同天数范围其他路线，最多 3 条 | ✅ | 待开发 |
| 6 | 路线详情 | 详情文案 + **daily_schedule JSON** + 餐饮/住宿/费用/标签 | ✅ | 待开发 |
| 7 | 关键词搜索 | 路线 `keyword` 模糊匹配 | ✅ | 待开发 |
| 8 | 反馈 | 文本入库；提交后提示成功 | ✅（legacy 未强制登录） | 待开发 |
| 9 | 种子数据 | 初始化脚本写入示例路线 | ✅ ~97 条 | 待开发（需改地域） |
| 10 | 统一导航 | 返回首页、登录态、Flash 提示 | ✅ | 待开发 |

**MVP 鉴权规则（legacy 实际行为，需在 PRD 确认是否保留）：**

| 页面 | legacy 是否需登录 |
|------|-------------------|
| 首页 | 否 |
| 注册 / 登录 | 否 |
| 智能推荐 / 搜索 / 路线详情 | **是** |
| 反馈 | **否**（与搜索不一致，重建时建议统一） |

**legacy 额外行为（可选 / 待删）：** `/search` 同时搜 **用户名**，产品书未提，MVP 可只做路线搜索。

### Wave 2 — AI + SEO（Web 组企划）

| # | 功能 | 负责人分工（原企划） |
|---|------|----------------------|
| 1 | 百度 **文心一言** API 封装 | 窦一曼：鉴权、重试、用量监控 |
| 2 | 智能对话推荐接口 | 同上 |
| 3 | 路线摘要生成接口 | 同上 |
| 4 | 反馈 **情感分析**接口 | 同上 |
| 5 | 路线向量嵌入（轻量中文模型） | 同上 |
| 6 | SEO：meta、JSON-LD、sitemap、robots | 袁瑞泽 |
| 7 | URL 语义化（ID → 关键词路径） | 袁瑞泽 |
| 8 | AI 对话组件前端 UI | 袁瑞泽 |
| 9 | 静态资源压缩、缓存 | 袁瑞泽 |

**项目书建议的 AI 落地策略（§8.1.3 技术风险）：**

- **双轨推荐**：规则匹配保底 + AI 辅助，避免冷启动无结果  
- 冷启动：热门路线 + 编辑精选  
- 渐进放量：先 5% 用户测 AI  

### Wave 3 — 竞赛远景（项目书有、MVP 不做）

- 一站式预订：门票 / 住宿 / 交通 / 特产电商 / 保险  
- AR/VR 导览、智能客服 7×24、客流预测、GIS 平台  
- Spark / MongoDB / Elasticsearch / TensorFlow Serving  
- 政府端监管中心、B2B 景区 SaaS、三阶段营收模型

## 5. 路线维度枚举

### 智能推荐表单（legacy 通用版）

**day_range：** `1-2` · `3-5` · `6-7`

**route_type：** 文化探索 · 山地风光 · 自然景观 · 美食之旅 · 亲子休闲 · 历史遗迹

**budget_level：** 经济型 · 舒适型 · 豪华型

### 地域数据注意

| 来源 | 数据倾向 | coding-dev 建议 |
|------|----------|-----------------|
| legacy `try` | 山西平遥等 | **不沿用**，与项目书山东定位冲突 |
| legacy 青岛版 | 即墨、金沙滩、海洋风光 | ✅ MVP 种子数据参考 |
| legacy 海大版 | 鱼山/浮山/崂山/西海岸校区 | 可选 **campus** 主题或二期 |
| 项目书 | 沿海 7 市、灵山岛、刘公岛等 | ✅ 叙事与命名主参考 |

**MVP 裁定：** 种子路线用 **山东沿海/青岛** 主题；`route_type` 中「山地风光」可改为项目书常用的 **海洋风光**（青岛版已如此）。

## 6. 页面与 UI（项目书 §3.3.2.2 + legacy 模板审计 2026-07-10）

**页面总数：7 个主页面 + 1 套全局壳**（顶栏 / Flash / 页脚）。完整出图 brief 见 `docs/output/reports/ui-prototype/ui-prototype-brief.md`。

| # | 页面 | legacy 模板 | 路由（重建建议） | 核心要素 |
|---|------|-------------|------------------|----------|
| P1 | 首页 | `index.html` | `/` | 智能推荐三下拉 + 「智能推荐」；快捷链：注册/登录/反馈 |
| P2 | 登录 | `login.html` | `/login` | 用户名/密码；链到注册 |
| P3 | 注册 | `register.html` | `/register` | 用户名/密码/确认密码 |
| P4 | 智能推荐结果 | `smart_search.html` | `/smart-search` | 路线卡 + 「查看详细行程」；相关推荐 ≤3；无结果态 |
| P5 | 关键词搜索 | `search.html` | `/search` | 仅路线列表（**去掉**「匹配的用户」） |
| P6 | 路线详情 | `route_detail.html` | `/routes/:id` | 概述、每日行程时间轴、饮食/住宿/费用、相关推荐 |
| P7 | 反馈 | `feedback.html` | `/feedback` | 多行文本 + 「提交反馈」 |

| 全局壳 | 要素 |
|--------|------|
| 顶栏 | Logo + **乡海云途**；未登录：注册/登录；已登录：欢迎 {用户名} + 注销 |
| 导航 | 内页左上「返回首页」；详情页「返回推荐」 |
| Flash | success / danger / info；约 3 秒自动消失 |
| 页脚 | 项目组署名（**不用** legacy「中北大学」） |
| 视觉 | 海洋蓝 `#0077BE` / `#00B4D8` + 海浪背景（**不用** legacy 山地绿主题） |

### legacy 与重建差异（已写死，防文档漂移）

| 项 | legacy 实际 | 重建裁定 |
|----|-------------|----------|
| 品牌文案 | 「乡旅与山e模式」 | **乡海云途** |
| 主色 | 山地绿渐变 | **海洋蓝**（项目书 §3.3.2.2） |
| 页脚 | 中北大学 | 中国海洋大学 / 乡海云途 Demo |
| `route_type` | 通用版「山地风光」 | **海洋风光**（青岛版已如此） |
| 关键词搜索入口 | **无**（仅 URL `?keyword=` 可达） | 顶栏或首页**必须补**搜索框 |
| 搜索页 → 详情 | 卡片**无**跳转链接 | **必须补**「查看详细行程」 |
| 搜索范围 | 路线 + 用户名 | **仅路线** |
| 反馈登录 | 不要求 | 暂保留不要求（§9 仍可 PRD 再议） |

现有素材：`assets/brand/` Logo · `assets/banners/` 背景
